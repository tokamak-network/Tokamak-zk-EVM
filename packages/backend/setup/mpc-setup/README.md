# MPC Ceremony Guide for Tokamak zk-EVM

This guide describes the current `mpc-setup` flow used by Tokamak zk-EVM.
It reflects the current implementation, not the older full-`x/y` phase-1 design.

## Overview

The current setup flow is split into:

1. Phase 1
   - An x-only ceremony that updates the phase-1 source contract consumed by phase 2.
2. Phase 2
   - A ceremony that prepares the final CRS, samples or reuses a concrete `y`, and then lets contributors update `\gamma`, `\delta`, and `\eta`.

`phase2_prepare` supports two phase-1 source modes:

- `native`
  - Use the Tokamak phase-1 accumulator generated by `phase1_initialize` and `phase1_next_contributor`.
- `dusk-groth16`
  - Skip the Tokamak phase-1 ceremony and instead use a Dusk Groth16 raw powers-of-tau challenge or response file as the phase-2 source.

The final phase-2 artifact format is unchanged across both modes.

## Prerequisites

Before running the ceremony:

- follow the repository prerequisites in the root project README
- ensure the frontend build artifacts are available
- install OpenSSL on your system if your platform requires it

All commands below assume:

```bash
cd "$PWD/packages/backend"
```

## Phase 1: Native Tokamak Mode

Use this mode when you want to run the Tokamak phase-1 ceremony before phase 2.

### Initialize phase 1

```bash
cargo run --release --bin phase1_initialize -- \
  --s-max 512 \
  --mode random \
  --setup-params-file setupParams.json \
  --outfolder ./setup/mpc-setup/output \
  --compress false
```

Important options:

- `--s-max`
  - Placement bound used by the setup.
- `--mode`
  - `random` or `beacon` in normal builds. In `testing-mode` builds, deterministic testing values are used automatically.
- `--setup-params-file`
  - Setup parameter JSON file.
- `--outfolder`
  - Output directory for ceremony artifacts.
- `--compress`
  - Whether JSON output stores compressed or uncompressed curve points.

### Add a phase-1 contributor

```bash
cargo run --release --bin phase1_next_contributor -- \
  --outfolder ./setup/mpc-setup/output \
  --mode random
```

### Batch-verify phase 1

```bash
cargo run --release --bin phase1_batch_verify -- \
  --outfolder ./setup/mpc-setup/output
```

## Phase 2: Native Source Mode

This mode consumes the latest Tokamak phase-1 accumulator.

### Prepare phase 2

```bash
cargo run --release --bin phase2_prepare -- \
  --outfolder ./setup/mpc-setup/output \
  --phase1-source-mode native
```

In native mode, `phase2_prepare` prompts for the last phase-1 contributor index and loads:

```text
phase1_acc_<index>.json
```

Optional arguments:

- `--mode random|beacon`
  - Controls how the initial phase-2 `y` is sampled when `--y-hex` is not supplied in normal builds.
  - In `testing-mode` builds, the phase-2 sampler is deterministic regardless of `--mode`.
- `--y-hex <HEX>`
  - Reuse an explicit `y` across multipart runs.
- `--total-part <N> --part-no <I>`
  - Split `phase2_prepare` into multiple parts.
- `--merge-parts true`
  - Merge previously generated part files into `phase2_acc_0.rkyv`.

### Add a phase-2 contributor

```bash
cargo run --release --bin phase2_next_contributor -- \
  --outfolder ./setup/mpc-setup/output \
  --mode random
```

### Batch-verify phase 2

```bash
cargo run --release --bin phase2_batch_verify -- \
  --outfolder ./setup/mpc-setup/output
```

### Generate final setup files

```bash
cargo run --release --bin phase2_gen_files -- \
  --intermediate ./setup/mpc-setup/output/intermediate \
  --output ./setup/mpc-setup/output/final
```

The generated outputs include:

- `combined_sigma.rkyv`
- `sigma_preprocess.rkyv`
- `sigma_verify.rkyv`

## Single-Contributor Wrappers

The orchestrator bins write intermediate ceremony files and final setup files to
different folders.

Native mode:

```bash
cargo run --release --bin native_mpc_setup -- \
  --subcircuit-library "$QAP_PATH" \
  --intermediate ./setup/mpc-setup/output/intermediate \
  --output ./setup/mpc-setup/output/final \
  --mode random
```

Dusk-backed mode:

```bash
cargo run --release --bin dusk_backed_mpc_setup -- \
  --subcircuit-library "$QAP_PATH" \
  --intermediate ./setup/mpc-setup/output/intermediate \
  --output ./setup/mpc-setup/output/final \
  --mode random
```

In both wrappers:

- the final output folder contains only the same three files written by `trusted-setup`
- the intermediate folder contains `phase1_acc_*`, `phase1_proof_*`, `phase2_acc_*`,
  `phase2_proof_*`, contributor info, and any downloaded Dusk raw response file
- `dusk_backed_mpc_setup` stores or downloads the Dusk raw response at `<intermediate>/dusk.response`

## Testing-Mode Builds

Like `trusted-setup`, `mpc-setup` uses the `testing-mode` cargo feature instead of a runtime
`testing` mode flag.

Examples:

```bash
cargo run --release --features testing-mode --bin native_mpc_setup -- \
  --subcircuit-library "$QAP_PATH" \
  --intermediate ./setup/mpc-setup/output/intermediate \
  --output ./setup/mpc-setup/output/final \
  --compress
```

```bash
cargo run --release --features testing-mode --bin dusk_backed_mpc_setup -- \
  --subcircuit-library "$QAP_PATH" \
  --intermediate ./setup/mpc-setup/output/intermediate \
  --output ./setup/mpc-setup/output/final
```

In `testing-mode` builds:

- phase 1 and phase 2 both use deterministic testing values automatically
- contributor prompts that exist only for normal random mode are skipped
- `--mode` still exists for interface compatibility, but no longer selects testing behavior

## Phase 2: Dusk-Backed Source Mode

This mode skips the Tokamak phase-1 ceremony and uses a Dusk Groth16 raw powers-of-tau artifact as the phase-2 source.

### Supported source artifact

`phase2_prepare` expects a raw Dusk Groth16 powers-of-tau challenge or response file, not a converted PLONK setup file.

Use:

```bash
cargo run --release --bin phase2_prepare -- \
  --outfolder ./setup/mpc-setup/output \
  --phase1-source-mode dusk-groth16 \
  --dusk-raw-file /path/to/dusk/raw/response
```

In this mode:

- Tokamak phase-1 contributor indexes are not used.
- The Dusk `\alpha` and `\beta` families are ignored.
- The Tokamak phase-2 source contract is read from plain Dusk tau powers under the identification described in:
  - `setup/mpc-setup/docs/phase2-output-plan.md`

After `phase2_prepare`, the remaining steps are the same:

- `phase2_next_contributor`
- `phase2_batch_verify`
- `phase2_gen_files`

## Current Ceremony Notes

- The current phase-1 contract is x-only.
- `phase2_prepare` is the first step that introduces a concrete `y`.
- The first phase-2 step discloses `y`, and later phase-2 contributors validate that the disclosed value is consistent and non-degenerate for the current setup.
- Downstream preprocess, prove, and verify flows continue to consume the same final artifact format.
